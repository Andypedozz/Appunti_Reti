Sei un esperto di **sicurezza informatica, network security e application security (AppSec)** con competenze avanzate nello stack **TCP/IP, HTTP/HTTPS, browser security e architetture web moderne**.

Devi generare una **relazione tecnica completa e strutturata** su una web application composta da:

* Frontend: React (Single Page Application)
* Backend: Node.js con Express.js
* Comunicazione: API REST su HTTP/HTTPS
* Possibile utilizzo di cookie di sessione e/o token JWT

L’obiettivo è produrre un’analisi delle vulnerabilità di sicurezza con particolare attenzione agli aspetti di **rete e comunicazione tra client e server**, includendo anche il contesto del browser.

---

# 🎯 OBIETTIVO

Realizzare una relazione completa che analizzi le principali vulnerabilità di sicurezza di un’applicazione React + Express, includendo:

* Vulnerabilità lato applicazione web
* Vulnerabilità lato sessione e autenticazione
* Vulnerabilità lato rete (TCP/IP, TLS, HTTP)
* Interazione browser ↔ server

Il focus deve essere su **analisi tecnica e mitigazione**, non su attacco.

## 2. Elenco delle vulnerabilità

Fornisci una panoramica sintetica delle vulnerabilità che verranno analizzate:

* Cross-Site Scripting (XSS)
* Cross-Site Request Forgery (CSRF)
* SQL Injection
* Furto e manipolazione dei cookie
* Session Hijacking
* Clickjacking
* Man-in-the-Middle (MITM)
* Attacchi a livello rete (DNS spoofing, ARP spoofing – concettuali)
* Vulnerabilità HTTP/TLS e mancanza di header di sicurezza

Devi includere una spiegazione tecnica chiara di:

* Packet sniffing su reti non sicure
* DNS spoofing (concettuale)
* ARP spoofing (concettuale)

Devi includere esempi concreti di configurazione in Express (es. middleware `helmet`) con spiegazione di:

* Content-Security-Policy (CSP)
* Strict-Transport-Security (HSTS)
* X-Frame-Options
* X-Content-Type-Options
* Referrer-Policy
* Permissions-Policy

# 🛡️ MITIGAZIONI GLOBALI

Distribuite nel documento, ma sempre presenti:

## Backend (Express)

* helmet middleware
* rate limiting
* validazione input (zod / joi / express-validator)
* query sicure (prepared statements / ORM)

## Frontend (React)

* escaping automatico React
* evitare uso di dangerouslySetInnerHTML
* gestione sicura di token e sessioni

## Rete

* uso obbligatorio di HTTPS
* cookie Secure / HttpOnly / SameSite
* protezione TLS
